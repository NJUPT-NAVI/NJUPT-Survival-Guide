## 一、科研绘图特征

科研配图为文字服务，需要与文字描述内容高度一致，视觉上一目了然，严谨且美观。

针对计算机方向科研论文中的图，笔者主观总结下来有 动机图（Motivation），框架流程图（Framework），数据分析图（Analysis），结果对比图（Comparison）四种。

- 动机图：为什么要做？描述现有方法存在的问题，并且与我们的方法进行对比 / 举几个例子描述一下论文的灵感来源（先验证明）
- 框架流程图：方法长啥样？精准的描述方法总体的框架，让审稿人理解方法的运作模式
- 数据分析图：为什么有效？通过图表来证明方法内部的机制是合理的，例如反应参数敏感性图，消融实验可视化图等
- 结果对比图：效果有多好？选择足够多和足够先进的方法和我们的方法进行可视化的对比

但无论是什么样的图，基本要满足 **布局饱满** ， **配色合适**，**格式一致** 三个特征。

### 1.1 布局饱满

:::note
**“ 拥有一个合理的布局已经成功一半了！(╹ڡ╹ )”**——笔者如是说。
:::


- **草稿理顺逻辑**
  
  一般在布局开始时，最好先想好这张图中所有的要体现的元素和逻辑，具体而言就是文字信息，图片信息，公式以及相互之间的逻辑。
  
  笔者个人习惯先将所有内容都先抽象成简单的形状和箭头表示出来，在草稿纸上绘制出它们之间的联系，此时不需要考虑美观性，重点在于：**这张图表达的意图是否能完整体现？整张图内部的逻辑是否合理？**

- **双栏or单栏？**
  
  论文一般都是左右两列，所以配图也一般是跨栏和单栏两种需求 （~~虽然也有非双栏的论文但是那种我没排过~~）。跨栏一般是横向长纵向短，单栏则相反。
  
  前面已经提到，配图服务于文字，因此要依据这张图在论文中的位置考虑好单栏排版还是双栏排版；如果不小心把纵向排的图放双栏或者双栏排的图放单栏就会发生字体超大或者超小的视觉悲剧。

- **对齐对称饱满**
  
  在草稿纸上画好大致的逻辑思路并且考虑好单双栏后，就可以打开绘图工具（具体使用见第二章节）进行操作了。
  
  笔者个人习惯先将所有要绘制的模块，文字，图片都先按照我们的草稿画出来放在界面上，因为是饱满布局，要是不小心漏了一个模块或者需要加上一个模块，后期调整起来是有点麻烦的。（~~实际写论文时这个概率非常高~~）
  
  根据纵向和横向的需求对图片的排版进行调整，调整过程中尽量最重要的是**对称和工整**。举个栗子，如果模块之间是相同结构，只有细节不同，可以对称绘制；如果是互相促进，可以考虑圆形，三角形，五/六边形这样封闭式的结构排版；

  **调整对齐尽量不要手调，又不够准确又多花时间，使用绘图工具的参考线和对其工具辅助！！**

- **美化微调布局**
  
  排版完所有的元素，可能难免会有部分位置感觉有所空缺；这时候在 [阿里巴巴图标库](https://www.iconfont.cn/) 上下载可以表达相应含义的icon图标，补充空缺，不仅补充了构图，更美化了整个页面。（例如表示重点可以用💡，🚩；表示效果的好坏可以用勾叉，笑脸哭脸等等）



### 1.2 配色合适

配色其实不单单是为了美观，本质上还是为了区分信息，引导读者视线来提升图片的可读性。主要有以下几个点要注意：

- **主色系，强调色**
  
  一张图中的大部分模块，框架，文字的颜色为主色，这类颜色最好不要有太高的饱和度；可以直接去使用 [配色网站1(colorspace)](https://mycolor.space/) , [配色网站2(pppalette)](https://www.fffuel.co/pppalette/) 或者找现成的配色方案抄他们的色号，偷懒一点的话直接用绘图工具自带的配色中比较浅的同系列颜色（如下图所示）；
  
  而强调色因为占比较少，可以使用正红（255，0，0）这样的饱和度超高的颜色来突出重点；

![配色参考](@assets/docs/learn/scientific/illustration/系列配色.png)


- **护眼配色**
  
  遵循“浅底深字、深底浅字”的原则，这是最基础也是最有效的护眼方式，杜绝高饱和亮色搭配、同色系浅底配浅字、深底配深字这类反人类配色，否则读者需要费力眯眼识别内容，大幅降低阅读效率。
  
  以下为错误示范 (图片源自网络)：
  
![糟糕配色示例](@assets/docs/learn/scientific/illustration/糟糕配色.png)

- **补救技巧**
  
  当然，有些特殊情况，例如必须标注在图片里的文字，会需要让浅色字在浅色底上显眼，可以采用文字背景填充透明度约50%的黑色的办法实现；反之，深色字在深色底可以填充一个透明度约50%的白色。
  
  这个操作对于内容很复杂的图片也有奇效。（如下图所示）

![图片文字示例](@assets/docs/learn/scientific/illustration/图片文字.png)


- **信息区分**
  
  使用颜色信息区分的级别有：画一个框然后写上文字（小模块不可再分）≤ 浅底色无边框模块组（多个小模块共用下层浅底色，形成整体区块）＝ 有色虚线框模块组

![颜色区分信息示例](@assets/docs/learn/scientific/illustration/颜色区分信息.png)

  


### 1.3 格式一致

高度统一的排版格式，才能让整体视觉更加规整，信息逻辑更清晰；虽然这个点非常繁琐，要非常有耐心去慢慢调整，但是它是提升整个画面的完成度的关键。

- **字体字号统一**

  统一层级的信息要用完全一致的字体字号；比如统一用12pt的新罗马字体加粗；

- **配色方案统一**

  同类模块用同样的色彩，即相同功能和相同层级的模块沿用一套配色逻辑；

- **细节元素统一**
  
  图中出现的所有小元素（如图标、箭头、装饰符号）应保持风格一致，例如：
  - 图标统一使用同一套风格（如线性、面性、手绘等），来源尽量一致；
  - 箭头样式统一（如所有箭头线宽均为1.5pt，箭头形状一致，弯曲弧度尽量相同；除非特殊箭头）；
  - 边框圆角统一（如所有模块圆角半径设为3pt或5pt，不可混用）；
  - 阴影效果统一（如需使用阴影，保持角度、模糊度、颜色一致）；
  - 元素间距统一（如模块之间的水平/垂直间距保持固定数值）。


## 二、科研绘图工具使用


### 2.1 Visio

:::note
本文使用Visio版本为专业版 2024
:::

Visio和PowerPoint，两者操作的界面的逻辑非常相似，但是Visio对于科研绘图这方面相对来说更加专业一点，Powerpoint操作简单一点；由于两者很多逻辑是相似的并且笔者最常用的是Visio，因此这里只进行Visio的使用介绍。

#### （1）常用快捷操作

| 功能分类 | 快捷键 | 功能说明 |
| :--- | :--- | :--- |
| **文件操作** | `Ctrl + N` | 新建绘图 |
| | `Ctrl + S` | 保存文件 |
| | `Ctrl + O` | 打开文件 |
| **视图控制** | `Ctrl + 鼠标滚轮` | 放大 / 缩小画布 |
| | `鼠标滚轮` | 上下移动 |
| | `Shift + 鼠标滚轮` | 左右移动 |
| **形状基础操作** | `Ctrl + C / V` | 复制 / 粘贴 |
| | `Ctrl + X` | 剪切 |
| | `Delete` | 删除选中形状 |
| | `Ctrl + Z` | 撤回上一步操作 |
| **形状排列** | `Ctrl + G` | 组合形状 |
| | `Ctrl + Shift + G` | 取消组合 |
| | `放大页面 + 方向键` | 微移形状（精细调整） |
| | `Ctrl + A` | 全选画布内容 |

| | `Ctrl + B` | 文本加粗 |


#### （2）快速排版技巧

- **排列和位置**: 

  这两个功能，通过鼠标拖动或者点击选取需要排版的组件，再选择对齐和相对位置关系就可以迅速让分散的组件变得干净利落。

  已经对齐好的可以对模块进行 **组合**，这样可以整体拖动模块的位置，调整起来非常方便。

  ![排版1](@assets/docs/learn/scientific/illustration/排版1.png)

- **参考线和对称**：

  很多流程图都有着沿着中心对称的结构，我们可以通过参考线和分布选项来完成，详见下图:

  另外，我们直接从工作界面边上的刻度线处可以分别用鼠标拖动出水平或者垂直的参考线；

  具体应用比如:把他们围成一个方框来迅速统一调整图像的大小，因为普通的调整可能是把图像重叠然后使用鼠标去微调让他们重合，但是参考线带有吸附的效果，不用太精细调整就会自动吸附到我们想要的位置。

  ![排版2](@assets/docs/learn/scientific/illustration/排版2.png)

#### （4）图形格式及公式插入

- **形状及格式**

  在绘制图像时候经常需要立体图形，特殊形状图像，或者专业的UML图组件，这种一般Visio有自带的形状，可以直接拖过来使用，不过为了更好地融入画面，需要进行相应的格式修正.

  格式修改除了最基本的线条和填充，还有进一步的三维旋转，阴影，映射等修改，可以进一步增加图像的立体感

  ![形状1](@assets/docs/learn/scientific/illustration/形状1.png)

  ![形状2](@assets/docs/learn/scientific/illustration/形状2.png)

- **公式插入编辑**

  公式插入的方式主要有两种：
  - 使用公式编辑器（例如mathtype）
  - 使用[在线latex公式编辑网站](https://www.latexlive.com/home)，生成透明底的公式图片

  在线网站生成可以使用图片识别公式功能，导出png格式。但是放入visio中放大可能会出现失真的问题，而公式编辑器产生的是矢量图，不会失真。

  后期公式需要进行修改，公式编辑器可以直接在visio内打开编辑器进行修改，而网站需要重新导出png。

  读者可根据以上利弊自行选择，以下仅展示mathtype的操作过程。


  ![公式1](@assets/docs/learn/scientific/illustration/公式1.png)

### 2.2 Origin

:::note
本文所用的Origin版本为2021，这一部分不太能系统整理知识，因为笔者也还在摸索阶段（。就从我目前接触到的内容慢慢撰写吧...以后要是学到新的方法可能还会再补充！
:::

Origin则用于绘制各种专业的可视化图表，相比于直接用程序画图，用这个工具可以通过修改数据的方式直接修改图表内容，非常适用于需要频繁调整的内容。



- **软件安装与破解**
  
  Origin破解版可上 [Filecr](https://filecr.com/windows/originpro/) 上下载，注意下载后导出图片时可能还有 **demo** 水印显示，可以下载该 [补丁](https://www.cnblogs.com/hui-rou/p/18303752) 。

- **数据导入**

  origin支持excel格式的复制黏贴，但是有些时候我们只有图片表格形式的数据，可以使用gpt之类的ai工具，识别给我们生成对应的excel格式或者csv格式，方便我们导入。

  ![origin1](@assets/docs/learn/scientific/illustration/origin1.png)

- **散点图/柱状图/雷达图绘制**
  
  由于Origin能画的图种类太多了，笔者也只绘制过其中几种比较常见的。如小标题这三种一起介绍是因为在origin中绘图逻辑是差不多的，通过设置原始数据的X,Y,Z轴适应不同图表的数据类别，可以迅速生成初始的图表。

  ![origin2](@assets/docs/learn/scientific/illustration/origin2.png)

  接下来就是具体的美化环节：
  - 双击图例的色块部分进入绘图细节面板，调整图的格式；
  - 双击坐标轴进入坐标轴面板调整坐标轴的范围，标签的字体字号等；
  - 右键图例空白部分点击设置图标，可以进行图例的格式调整。

  **这里的刻度设置有个小技巧：如果同一组数据之间数据差太大，太小的数据在图中难以辨别，我们可以修改刻度中的类型改成对数型，可以显著缩小显示的差距，对应的刻度也会发生改变！**

  ![origin3](@assets/docs/learn/scientific/illustration/origin3.png)

- **直方图的绘制**
  
  直方图虽然也可以通过上述方式直接生成，不过这种形式没有统计后的数据，只有统计后的图表；因此我们可以采用先通过统计原始数据的频数，再对统计后的图像生成图像的方式来绘制直方图。

  ![origin4](@assets/docs/learn/scientific/illustration/origin4.png)

- **图表导出**
  
  首先可以通过菜单栏中的导出选项导出png或者pdf格式，很简单不赘述。

  其次更常用的是复制页面到viso软件，类似mathtype公式那样是可编辑的形式，在visio中双击可打开项目。同样放大不会失真，而且是透明底，一般都是用这个方法导出。

  ![origin5](@assets/docs/learn/scientific/illustration/origin5.png)
